# Feature: Native Platform Showcase

**Date:** 2026-06-08

Generated by `bash tool/scaffold_feature_contract.sh --name native_platform_showcase --apply`.

## Problem

Interviewers and learners need a concise, cross-platform explanation of how Flutter embeds native views, packages, FFI, graphics, and gestures—and a **live** proof that Dart can call Swift, Kotlin, and C/C++ on supported hosts.

## Scope

- In: Capability catalog, four lesson cards, **live interop** (Swift/Kotlin MethodChannel + C/C++ FFI), platform-adaptive UI, Example page entry, route + deep link, l10n, tests, feature README, smoke integration flow, web preflight smoke
- Out: Replacing `apps/mobile/lib/shared/platform/native_platform_service.dart` (separate channel for runtime platform info)

## Layers Touched

- [x] domain (`NativeInteropCallResult`, platform service ports, `LoadNativePlatformShowcaseUseCase`)
- [x] data (`MethodChannelNativeShowcaseHostLanguageService`, `FfiNativeShowcaseNativeCodeService`, `NativePlatformInfoRepositoryImpl`)
- [x] presentation (interop section UI; cubit → use case only)

## Architecture

Educational stack: **presentation → use case → repository → platform services → interop adapters → host code**. UI and cubit never import `MethodChannel`, `dart:ffi`, or generated bindings.

```text
NativePlatformShowcaseCubit
        ↓ LoadNativePlatformShowcaseUseCase
NativePlatformInfoRepository.loadShowcase()
        ↓
NativeShowcaseHostLanguageService  →  MethodChannelNativeShowcaseHostLanguageService  →  Swift / Kotlin
NativeShowcaseNativeCodeService    →  FfiNativeShowcaseNativeCodeService            →  native_showcase.c
```

| Concern | Owner |
| --- | --- |
| Route + cubit wiring | `apps/mobile/lib/app/router/routes_demos.part.dart` (`createNativePlatformShowcaseRoute`) |
| DI | `apps/mobile/lib/core/di/features/register_native_platform_showcase_services.dart` |
| Live interop merge | `NativePlatformInfoRepositoryImpl` (catalog via `mapShowcase` + three interop calls) |
| Runtime platform label | `RuntimePlatformProbe` in data layer (not shared `NativePlatformService`) |

**Out of scope:** `apps/mobile/lib/shared/platform/native_platform_service.dart` — separate channel for app-wide platform info; showcase uses its own channel for demo interop only.

Canonical detail: [`apps/mobile/lib/features/native_platform_showcase/README.md`](../../apps/mobile/lib/features/native_platform_showcase/README.md).

## Contracts

- Flow: `LoadNativePlatformShowcaseUseCase` → `NativePlatformInfoRepository.loadShowcase()` → platform services
- Repository: `loadShowcase()` → `PlatformShowcaseData` with `interopResults`
- Host languages: `NativeShowcaseHostLanguageService` (MethodChannel impl today; swappable to JNI / SwiftGen)
- Native code: `NativeShowcaseNativeCodeService` (FFI impl today)
- Host channel: `com.example.flutter_bloc_app/native_showcase` — `invokeSwift`, `invokeKotlin`
- FFI: `native_showcase_greeting`, `native_showcase_add` in `native/native_showcase/`
- Probe: `RuntimePlatformProbe({this.isWeb, this.platform})` → `AppPlatformKind` via `resolve()`
- State: Freezed union `initial` / `loading` / `loaded` / `error` with `NativePlatformShowcaseFailureKind`
- Mapper: `platformCapabilityCatalog` + `mapShowcase(AppPlatformKind)`; order from `showcaseCapabilityOrder`
- Test catalog-only repo: `SimulatedNativePlatformInfoRepository` (no interop; mapper tests)
