# Brainstorming

Use when request ambiguous, multi-approach, product/design tradeoffs.

Triggers (examples):

- “brainstorm”, “ideas”, “options”, “approaches”, “tradeoffs”, “which should we pick”
- “not sure”, “unclear requirements”, “help me decide”

Canon first:

- `AGENTS.md`
- `docs/agent_knowledge_base.md`
- `docs/agents_quick_reference.md`

Hard gates:

- Don’t edit until >=95% confidence in goal/scope/approach.
- If unclear, ask 1–2 critical questions then proceed.

Output shape (keep short):

- State goal + constraints + success criteria.
- List 2–3 approaches with tradeoffs; pick lowest-regret.
- Identify smallest verifiable slice + proof step.
